\section{Recherche par Voisinage Variable (VNS)}

La méthode VNS (\textit{Variable Neighborhood Search}) est une métaheuristique qui explore systématiquement plusieurs structures de voisinage pour échapper aux optima locaux. L'idée clé est qu'un optimum local pour un voisinage ne l'est pas nécessairement pour un autre.

\subsection{Objectif et solution initiale}

L'objectif est de minimiser $\sum T_j$ avec $T_j = \max(0, C_j - D_j)$.

On part de la solution obtenue par l'heuristique LTR, qui donne $\sum T_j = 5$ :
\begin{itemize}
    \item $C_1 = 8$, $T_1 = \max(0, 8-6) = 2$
    \item $C_2 = 7$, $T_2 = \max(0, 7-8) = 0$
    \item $C_3 = 8$, $T_3 = \max(0, 8-5) = 3$
\end{itemize}

\textit{Remarque :} Le Job 3 a un temps total de traitement de 8 et une échéance $D_3=5$, donc $T_3 \geq 3$ est incompressible. La borne inférieure de $\sum T_j$ est donc 3.

\subsection{Structures de voisinage}

On définit trois voisinages de taille croissante :

\begin{enumerate}
    \item[\textbf{$N_1$}] \textbf{Swap intra-machine :} permuter deux opérations consécutives sur une même machine.
    \item[\textbf{$N_2$}] \textbf{Swap inter-machine :} pour un même job, échanger l'ordre de passage entre deux machines (modifier la séquence d'opérations du job).
    \item[\textbf{$N_3$}] \textbf{Réinsertion :} déplacer une opération à une autre position sur sa machine.
\end{enumerate}

\subsection{Algorithme}

\begin{enumerate}
    \item Choisir une solution initiale $S_0$ (ici, la solution LTR).
    \item Poser $k = 1$ (indice du voisinage courant).
    \item \textbf{Perturbation :} générer un voisin $S'$ aléatoire dans $N_k(S)$.
    \item \textbf{Recherche locale :} améliorer $S'$ par descente dans $N_1$ pour obtenir $S''$.
    \item \textbf{Mise à jour :} si $\sum T_j(S'') < \sum T_j(S)$, alors $S \leftarrow S''$ et $k \leftarrow 1$. Sinon, $k \leftarrow k+1$.
    \item Répéter jusqu'à $k > 3$ ou critère d'arrêt atteint.
\end{enumerate}

\subsubsection{Itération 1 --- Voisinage $N_1$ (swap intra-machine)}

On permute $J_2$ et $J_1$ sur $M_1$ (J1 passait avant J2). On obtient : sur $M_1$, on place d'abord $J_2$ (durée 1) puis $J_3$ (durée 2) puis $J_1$ (durée 3). On réordonne aussi $M_2$ et $M_3$ pour respecter les contraintes de non-chevauchement.

Nouvel ordonnancement :
\begin{itemize}
    \item $M_1$ : $J_2$ [1], $J_3$ [2--3], $J_1$ [4--6]
    \item $M_2$ : $J_3$ [1], $J_1$ [2--3], $J_2$ [4--7]
    \item $M_3$ : $J_1$ [1], $J_2$ [2--3], $J_3$ [4--8]
\end{itemize}

Calcul : $C_1 = 6$, $T_1 = 0$ ; $C_2 = 7$, $T_2 = 0$ ; $C_3 = 8$, $T_3 = 3$.

$\sum T_j = 3 < 5$ : \textbf{amélioration !} On accepte cette solution et on remet $k = 1$.

\subsubsection{Itérations 2 à 4 --- Exploration des trois voisinages}

À partir de cette solution ($\sum T_j = 3$), on explore successivement :
\begin{itemize}
    \item \textbf{$N_1$ :} Les swaps intra-machine testés ne produisent aucune amélioration. $k \leftarrow 2$.
    \item \textbf{$N_2$ :} Les échanges inter-machine pour chaque job ne réduisent pas $\sum T_j$. $k \leftarrow 3$.
    \item \textbf{$N_3$ :} Les réinsertions d'opérations ne permettent pas non plus d'améliorer. $k \leftarrow 4 > 3$.
\end{itemize}

\textbf{Arrêt :} tous les voisinages ont été explorés sans amélioration.
